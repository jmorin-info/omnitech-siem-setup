## Ecarts a traiter (par priorite)

1. ❌ **Externalisation des preuves (5.33, 8.13)** — les sauvegardes restent sur
   l'hote (`/home/siem-backup`). Un attaquant qui prend la VM efface tout.
   → Action : rsync quotidien vers le NAS (ou repo S3) **en push avec compte
   dedie en ecriture seule** ; idealement immutabilite (snapshot NAS).
2. ❌ **Test de restauration documente (8.13)** — la restauration n'a jamais
   ete rejouee. → Action : exercice trimestriel : restaurer mongodump + 1
   snapshot sur une VM jetable, consigner duree/resultat (preuve d'audit).
3. 🟡 **Tracabilite des actions sur le SIEM (8.15)** — les actions admin dans
   Graylog ne sont tracees que par nginx access.log (l'audit log applicatif
   est une fonction Enterprise). → Action : conserver/exporter
   `/var/log/nginx/access.log` (90 j) + `auditd` sur la VM ; proscrire le
   compte admin partage (creer des comptes nominatifs).
4. 🟡 **Comptes nominatifs + MFA (5.16, 5.17)** — `admin` est partage et sans
   MFA. → Action : 1 compte nominatif par admin ; en façade, pre-authentification
   possible via nginx (mTLS certificats PKI) si exigence forte.
5. 🟡 **Couverture des sources (8.15)** — manquent : DHCP (filebeat sur les
   logs DHCP des DC), DNS Server (audit analytique), sauvegardes Veeam
   (events), ESXi/vCenter (syslog 1514 deja pret cote Graylog).
6. 🟡 **Procedure de reponse (5.26)** — les alertes partent mail+Teams mais
   sans procedure ecrite (qui qualifie, en combien de temps, escalade).
   → Action : runbook 1 page par famille d'alerte (P3 = 1 h ouvree, etc.).
7. 🟡 **Revue periodique (5.36)** — le rapport hebdo existe ; formaliser une
   revue mensuelle (faux positifs, seuils, nouvelles detections) avec trace.
8. ❌ **Secret M365 et certificats : gestion du cycle de vie (5.37)** — secret
   client Entra (24 mois) et cert PKI (2028-06-10) expireront silencieusement.
   → Action : echeances au calendrier + alerte deadman (le silence du
   collecteur M365 declenchera "0 message" — ajouter une alerte dediee).
9. 🟡 **Durcissement VM (8.9)** — base Debian saine (services localhost,
   nftables) ; completer : MAJ automatiques de securite (unattended-upgrades),
   auditd, bannieres SSH, cle SSH plutot que mot de passe root.

## Argumentaire retention (a inscrire dans la politique)
- Windows Security / FortiGate : 90 j en ligne (investigation courante)
- Sysmon / autres canaux : 60 j (volumetrie)
- M365 : 180 j (tracabilite cloud, pas d'autre copie interne)
- Long terme : snapshots quotidiens, 14 j locaux + export NAS a mettre en place
  (cible recommandee : 1 an pour les preuves d'authentification).
