# Microsoft 365 → Graylog — plan d'integration (etape suivante)

Objectif : connexions suspectes Entra ID, sign-ins risques, audit Exchange/
SharePoint/Teams, le tout normalise dans Graylog (stream + dashboard + alertes).

## Architecture retenue (Graylog OSS, sans plugin payant)

```
Entra ID / M365 ──Graph API / O365 Management Activity API──┐
                                                            ▼
              fetcher Python (cron 5 min, sur la VM SIEM, etat persistant)
                                                            ▼
              input GELF HTTP 127.0.0.1:12201 ──► stream "OMNI - M365"
              (pipeline : normalisation user/src_ip/action + GeoIP + alertes)
```

## Ce qu'on collectera

| Source API | Contenu | Valeur |
|---|---|---|
| Graph `auditLogs/signIns` | connexions (IP, pays, MFA, appareil, echec/succes) | impossible travel, pays inhabituel, spray |
| Graph `identityProtection/riskyUsers` + `riskDetections` | scoring Microsoft | reprise directe en alertes P3 |
| Graph `auditLogs/directoryAudits` | modifs Entra (roles, apps, MFA reset) | sabotage / persistance cloud |
| O365 Mgmt Activity `Audit.Exchange` / `Audit.SharePoint` | regles boites mail, partages, DLP | exfiltration, regles de transfert malveillantes |

## Pre-requis a faire cote tenant (toi, ~15 min)

1. **Entra ID > App registrations > New** : `OMNI-SIEM-Collector`.
2. **API permissions** (type *Application*, puis « Grant admin consent ») :
   - Microsoft Graph : `AuditLog.Read.All`, `IdentityRiskyUser.Read.All`,
     `IdentityRiskEvent.Read.All`, `Directory.Read.All`
   - Office 365 Management APIs : `ActivityFeed.Read`
3. **Certificates & secrets** : creer un client secret (24 mois max).
4. Me fournir (a mettre dans `00-vars.env`) :
   ```
   M365_TENANT_ID='...'
   M365_CLIENT_ID='...'
   M365_CLIENT_SECRET='...'
   ```
5. Verifier que la VM SIEM sort en HTTPS vers `login.microsoftonline.com`,
   `graph.microsoft.com`, `manage.office.com` (policy FortiGate VLAN 220 -> WAN).

## Ce que je ferai ensuite (scripts 16-17, deja cadres)

- `16-m365-input.sh` : input GELF HTTP (127.0.0.1:12201), index set
  `omni-m365` (180 j - traçabilite cloud), stream + pipeline (normalisation
  vers le schema OMNI : `user`, `src_ip`, `event_action`, `alert_tag`).
- `17-m365-fetcher.sh` : deploie `/usr/local/sbin/omni-m365-fetch.py`
  (curseurs persistants dans /var/lib/omni-m365, dedup, backoff 429)
  + timer systemd 5 min.
- Alertes : `OMNI - M365 connexion a risque (Identity Protection)`,
  `OMNI - M365 connexion pays inhabituel`, `OMNI - M365 regle de transfert
  mail creee`, `OMNI - M365 role privilegie attribue`.
- Dashboard `OMNI - Microsoft 365` : carte des connexions, echecs MFA,
  utilisateurs a risque, activite admin.
